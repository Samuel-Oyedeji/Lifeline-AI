1. High-Level System Architecture

The React dispatch console and plain HTML/JS ambulance driver page communicate with the FastAPI backend over WebSocket and REST. The backend calls OSRM (routing) and Overpass (hospitals) via httpx, and keeps ambulance + incident state in memory.

4. Data Flow #2: Pickup to Hospital

The hospital pipeline: discover (Overpass) → pre-rank (OSRM Table, top 3) → route each finalist → decide by effective time → security recommendation.

6. Deployment Architecture (AWS EC2 + S3)

The static frontend (React console build + ambulance driver page) is hosted on AWS S3 (static website, optionally fronted by CloudFront). The FastAPI backend runs on an AWS EC2 instance via Uvicorn. Browsers load the UI from S3 and connect to EC2 over HTTPS/WSS for REST + WebSocket; EC2 calls external OSRM and Overpass via httpx.
